## 为什么

Jujutsu (jj) 是一个强大的 Git 兼容版本控制系统，拥有现代化的 CLI 界面，但它缺少图形用户界面。偏好可视化版本控制交互的用户——尤其是从基于 GUI 的 Git 客户端（如 GitHub Desktop、Sourcetree 或 GitKraken）迁移过来的用户——面临着陡峭的学习曲线。一个 GUI 将使 Jujutsu 对更广泛的受众更加易用，提高其独特功能（如操作日志、不可变提交和冲突解决）的可发现性，并提供更直观的工作流可视化。

## 变更内容

此变更引入了一个基于 Web 的 Jujutsu GUI 应用程序：

- **新 GUI 应用程序**：用于与 Jujutsu 仓库交互的可视化界面
- **仓库可视化**：交互式变更视图
- **变更管理**：可视化创建、编辑、放弃和移动变更
- **命令面板**：通过键盘快捷键快速访问常用 jj 命令

## 能力

### 新增能力

- `repo-management`：仓库初始化、克隆和基本仓库操作（打开、关闭、切换）
- `commit-graph`：提交历史的可视化表示，包括分支可视化、书签管理和变更导航
- `change-operations`：创建新变更、编辑提交消息、放弃变更，可视化移动/压缩变更
- `working-copy`：工作副本状态的可视化表示、文件暂存和丢弃操作

### 修改的能力

_无 — 这是一个新项目，没有现有能力需要修改。_

## 影响

**新应用程序**：这创建了一个全新的 Web 应用程序，包含以下组件：

- **前端**：基于 Web 的 UI（可能使用现代框架如 React/Vue/Svelte）
- **后端**：与 jj CLI 命令交互的 API 服务器
- **集成层**：GUI 与 jj 命令行界面之间的桥接层

**依赖项**：
- 必须安装 Jujutsu CLI (`jj`) 并在 PATH 中可用
- Web 应用程序需要 Node.js 运行时
- 文件系统访问可能需要原生集成

**受影响的系统**：
- 仓库根目录下的新项目结构
- 构建和开发工具配置
- GUI 应用程序的 CI/CD 流水线
